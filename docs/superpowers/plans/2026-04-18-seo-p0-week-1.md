## Task 1: H1 estático en `/inteligencia-artificial`

**Problema:** `app/inteligencia-artificial/sections/HeroSection.tsx:86-98` usa `useState('')` para `displayed` y anima el H1 vía typewriter. En SSR, el H1 se renderiza con `{displayed}` = `''` → Google indexa un H1 vacío.

**Solución:** renderizar el texto completo como fallback estático dentro del `<h1>` visible para crawlers y bots que no ejecutan JS, y superponer la animación sobre él con CSS. Alternativa más simple y menos invasiva: añadir un `<span className="sr-only">` con el texto completo dentro del H1, accesible para crawlers y screen readers; la animación visual sigue controlando el texto visible.

**Files:**
- Modify: `app/inteligencia-artificial/sections/HeroSection.tsx:86-98`

- [ ] **Step 1: Leer el estado actual del H1**

Abrir `app/inteligencia-artificial/sections/HeroSection.tsx` y localizar el bloque líneas 86-98.

- [ ] **Step 2: Añadir fallback SSR accesible dentro del `<h1>`**

Reemplazar el bloque actual:

```tsx
<h1 className="font-display text-display-lg md:text-display-xl lg:text-[80px] lg:leading-[0.97] lg:font-normal text-text-on-dark max-w-5xl">
  {displayed}
  {(phase === 'retyping' || phase === 'done' || phase === 'waiting') && corrected.length > 0 && (
    <span className="text-rl-red">{corrected}</span>
  )}
  {phase !== 'done' && phase !== 'waiting' && (
    <motion.span
      className="inline-block w-[3px] h-[0.75em] bg-rl-red ml-1 align-middle"
      animate={{ opacity: [1, 0] }}
      transition={{ duration: 0.6, repeat: Infinity, repeatType: 'reverse' }}
    />
  )}
</h1>
```

por:

```tsx
<h1 className="font-display text-display-lg md:text-display-xl lg:text-[80px] lg:leading-[0.97] lg:font-normal text-text-on-dark max-w-5xl">
  {/* SEO/a11y fallback — texto final completo para crawlers y lectores de pantalla */}
  <span className="sr-only">
    La inteligencia artificial amplifica al experto.
  </span>

  <span aria-hidden="true">
    {displayed}

      <span className="text-rl-red">{corrected}</span>

</h1>
```

## Task 3: Google Tag Manager (GA4 via GTM)

**Problema:** sin medición. `.env.local.example` ya reserva `NEXT_PUBLIC_GTM_ID` pero nunca se conectó.

**Solución:** instalar GTM (no GA4 directo — GTM permite añadir Pixel, Clarity, conversions después sin re-deploy). Carga condicional sobre presencia de `NEXT_PUBLIC_GTM_ID` para no romper previews sin env var. Usar `next/script` con strategy `afterInteractive` para no bloquear LCP.

**Prerequisito manual del usuario:** crear contenedor en tagmanager.google.com, copiar el ID `GTM-XXXXXXX`, luego en el contenedor crear un tag GA4 con el `Measurement ID` que le obtendremos creando también una propiedad GA4 en analytics.google.com. Este plan asume que ya tienes el GTM ID antes de ejecutar Step 6 — si no, ejecuta Steps 1-5 y pausa hasta tenerlo.

**Files:**
- Create: `components/analytics/GoogleTagManager.tsx`
- Modify: `app/layout.tsx`
- Modify: `.env.local`

- [ ] **Step 1: Crear componente GTM**

Crear `components/analytics/GoogleTagManager.tsx`:

```tsx
import Script from 'next/script'

interface Props {
  gtmId: string
}

export default function GoogleTagManager({ gtmId }: Props) {
  return (
    <Script
      id="gtm-script"
      strategy="afterInteractive"
      dangerouslySetInnerHTML={{
        __html: `
(function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start':
new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0],
j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src=
'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f);
})(window,document,'script','dataLayer','${gtmId}');
        `,
      }}
    />
  )
}

export function GoogleTagManagerNoScript({ gtmId }: Props) {
  return (
    <noscript>
      <iframe
        src={`https://www.googletagmanager.com/ns.html?id=${gtmId}`}
        height="0"
        width="0"
        style={{ display: 'none', visibility: 'hidden' }}
      />
    </noscript>
  )
}
```

## Task 4: Redirect 301 www → non-www

**Problema:** `https://www.redesignlab.org/` y `https://redesignlab.org/` ambos devuelven 200, con HTML idéntico. El canonical señala non-www, pero es solo un hint. Debería ser un redirect 301 duro.

**Solución:** usar `redirects()` en `next.config.mjs` con `has: [{ type: 'host', value: 'www.redesignlab.org' }]`. Esto funciona en Vercel.

**Files:**
- Modify: `next.config.mjs`

- [ ] **Step 1: Reemplazar `next.config.mjs`**

Actual:
```js
/** @type {import('next').NextConfig} */
const nextConfig = {};

export default nextConfig;
```

Reemplazar por:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  async redirects() {
    return [
      {
        source: '/:path*',
        has: [{ type: 'host', value: 'www.redesignlab.org' }],
        destination: 'https://redesignlab.org/:path*',
        permanent: true,
      },
    ]
  },
}

export default nextConfig
```

- [ ] **Step 2: Build**

```bash
cd redesignlab && npm run build
```

Expected: compila sin errores. `next build` valida la sintaxis de `redirects()`.

- [ ] **Step 3: Verificar en preview/producción (post-deploy)**

```bash
curl -sI https://www.redesignlab.org/ | head -5
```

Expected:
```
HTTP/2 308   # o 301 (Next.js emite 308 por defecto con permanent: true)

```

Nota: Next.js + Vercel emiten **308 Permanent Redirect** (equivalente moderno de 301 que preserva método HTTP). Google los trata igual.
